Wii Tanks Reinforment Learning Agent - Summer 2023

Tanks is a minigame featured in Nintendo’s Wii Play video game. Using Dolphin (a Wii and Gamecube emulator for computers), the game can be played on a computer along with emulated controls. With this in mind, the goal of this project was to create a bot in Python that would be able to complete the various levels of the game.
The first approach was using PyAutoGUI and OpenCV. OpenCV was used so the bot can “see” everything within the game while PyAutoGUI was used to control the game. A cascade classifier was trained using 150 images of the Player’s tank. After training, the classifier was able to recognize all the various types of tanks, but not differentiate between them. So the player tank and an enemy tank were the same thing in the eyes of the bot. In order to get around this, color masks were drawn around specific colors. The player’s tank is blue, so everything but blue was masked, the classifier recognized the tank and it was labeled appropriately. Inversely, blue was masked, and the classifier recognized all the enemy tanks. This approach worked successfully, but I ended up pivoting away from it in order to use a neural network.
The second approach was using the Stable-Baselines3 library. This library has lots of built in functions to train reinforcement learning models. A custom environment was created for the bot and a reward system was implemented. The rewards system was solely focused on eliminating enemy tanks, as time alive would just encourage the bot to stay alive, and not actively eliminate tanks. Using OpenCV again, the matchTemplate() function was used to recognize the various numbers in the specific score font. Simple image to text couldn’t be used since it was not trained on that specific font and would output erratic numbers.   After successfully im plementing the environment and reward function, the bot would play the game unsuccessfully. After training f or ~20000 time steps, the bot progressed to the second level but not consistently. More training has to be done.

Magstripe Card Swiper - Summer 2023

This is a fully functioning magstripe ID card reading system. It uses Python and Excel sheets to sign in and out students when they swipe their card. The GUI was made using Python's Tkinter library.

The program's main feature is to sign in and out, capturing the date and time at which it was done, but also includes many more features. Since it is tailored to the inVenTs Studio at Virginia Tech, signing in will bring up user data such as what tools they are currently trained on. It also features a button in which it will sign out all the students at closing time. New users are asked to fill in information to be added to the existing database.

The OpenPyxl library and Pandas were used in conjunction to work with spreadsheets, pulling and modifying data as necessary. Spreadsheets were used to easily export and handle data for statistics purposes.

The repo for this project can be found here.

CNC Sand Coffee Table - Summer 2022

Inspired by the Sisyphus Sand Table and the ZenXY, this sand table was created using an old glass table top and spare wood.  3D printing software, Klipper, was flashed to an Arduino Uno, and is able to be controlled through a Raspberry Pi.  Images in the sand can be generated through sandify.org, which users can hand create various line works, similar to vector drawing programs.  To control the LEDs in the table, WLED is used for network control and Home Assistant integration.

In order to match the curved glasstop, the corners were measured and 3D printed to get the exact curve.  The wood covering below the glass was lasercut to exact size.  The legs and sidings were all hand cut with a jigsaw.  Afterwards it was painted accordingly.

The sand table pictured is revision two.  The first version used linear rails and a sub optimal belt path which caused it to misalign itself after drawing pictures in the sand.

Laser Engraved LED Picture Frame - Fall 2022 

Originally starting as simple laser engraved wooden posters, it evolved to become a picture frame.  The original artwork is hand traced in Inkscape in order to get a vectorized image of the poster.  That image is laser engraved onto an 11" x 11" piece of wood.  The image is then mirrored and laser engraved again onto an 11" x 11" piece of acrylic.  The acrylic is mirrored so when it is turned around and placed on top of the wood, you cannot feel the engraved lines.

The width of the wood and acrylic were measured and a frame was 3D modeled in Fusion 360.  The frame is printed in four pieces along with four corners, all printed upright to avoid overhangs.  An LED strip is strung through the frame and dupont connectors are crimped to the LED strip in order for it to be easily rearranged for a college dorm room.  A custom PCB was CNC'd out of a single layer copper FR4 board.  This was done to make wiring neater and as a proof of concept.

With the LEDs off, the wood engraving is shown.  But with the LEDs on, the acrylic engraving is lit, and shows off the detailed linework that was engraved.

Infrared Radio Teletype Project - Spring 2023

During my Sophomore Spring semester, I took Virginia Tech's ECE Sophomore design class. The project I chose to do was this Infrared Radio Teletype project. The basic idea of the project was to send and receive messages only using an IR LED, an IR photodiode, and two Arduinos. I was able to achieve sending and receiving a  message over 30 ft.

Using Arduino C, I programmed a method of decoding a string into 8-bit binary (ASCII representation) with a start and stop bit for each letter. This program would take a string, break it down into letters and then each letter would get broken down into 10 bits. The IR LED would then be turned on and off at specific frequencies depending if it was a 1 (2125 Hz) or a 0 (2295 Hz) (These frequencies were project spec).

On the receiving end, a transimpedance amplifier was used to convert the photodiode's current into a readable voltage. This voltage was then passed through a series of bandpass filters to reduce noise and to single out either a 1 or 0. The receiving Arduino used a digital comparator to compare whether the receiving signal was noise or the real deal. If it was the real deal, the frequency was measured appropriately using the input capture registers and assigned to a 1 or 0. The program then worked nearly identical to the sending side, but in reverse.

To go above and beyond, I built a T-9 dialing device that would write a message on an OLED screen and send the message. I ended with an A in the class and a report on this project is available on request.

Simple Computer Project - Fall 2022

Over the course of my Sophomore Fall semester, my Digital Systems class gave us a DE10-Lite development board to develop and test our systems on.  The class consisted of learning how to use Verilog in order to create parts of a simple computer.  It had us developing arithmetic, logic, and shift units using just logic gates, rather than using bitwise operators. In addition to creating these compute units, we developed a register and bus system to move the data around within our simple computer.  This class was foundational and each system tended to build off of previous work.  In addition to learning how to create these systems, we were taught to minimize our transistor count and to create systems efficiently.

By the end of the semester, we brought all our systems together into a simple computer.  It took some tinkering to have all the systems work well with each other, but I was successful, earning an A in the class.  A report on the function unit, instruction decoder, and register system is available upon request.

Voron 2.4 FFF 3D Printer - Fall 2022 

This Voron 2.4 (serial 4790) was self-sourced and built over the course of a couple of weeks.  All the 3D printed parts were printed on a Prusa MK3S+ and the wiring was hand crimped to exact sizing as needed.  It has the following mods:  Klicky, Klipperscreen, chamber lights, chamber thermistor, THE FILTER, and a CAN Bus toolhead.

I designed a filament sensor and ECAS fitting adapter for the toolhead, lacking in the original Stealthburner design.  This filament sensor uses an Omron D2F switch that is pushed when filament is inserted and auto-loads filament similar to the Prusa MK3S+.

The CAN Bus toolhead is the SB2040 flashed with CanBoot and using a UTOC connected to the Raspberry Pi.  The umbilical wire was heat shrinked and cut to size as needed.  The strain relief is a piece of nylon filament holding up the umbilical wire.

This printer acts as my workhorse printer and pumps out perfectly dimensioned parts as needed for projects.

Prusa MK3 -> MK3S+ & MMU2S - Summer 2022

Originally a MK3, this 3D printer was my first printer.  In the Summer of 2022, I upgraded it to a MK3S+ with a Phaetus Dragon hotend.  The original orange parts were replaced with yellow ABS parts in order to sustain higher temperatures.  Along with upgrading it to a MK3S+, a camera and Octoprint were added.  This was done to add a locally hosted Obico server to watch for 3D printing mishaps.

Along with upgrading to a MK3S+, the MMU2S upgrade was added.  This brought multi material printing to this single toolhead printer and allowed for me to print in various colors and water soluble supports for easier cleanup.  While difficult to use at first, it was eventually tinkered out to provide near perfect prints.
